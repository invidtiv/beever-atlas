# Roadmap

## v0.2.0 - Community Feedback (Q2 2026)
- Interactive API playground in docs
- Demo GIF and video tutorials
- Custom domain deployment guide
- Chinese and Japanese documentation
- Community-contributed adapters

## v0.3.0 - Enterprise Features (Q3 2026)
- SSO/SAML authentication
- Audit logging and compliance
- Multi-tenant support
- Role-based access control (RBAC)
- Custom embedding models

## v1.0.0 - Stable Release (Q4 2026)
- Stable API with versioning
- Guaranteed backward compatibility
- Performance benchmarks and SLAs
- Production deployment guides for cloud providers
- Official Docker images on GitHub Container Registry

## v1.1.0 - Advanced Features (Q1 2027)
- Real-time collaboration on wiki pages
- Custom extraction schemas
- Advanced graph analytics
- API key management
- Webhook integrations

## Community Requests

We prioritize features based on community feedback. To suggest features:

1. **Open a GitHub Discussion** - Start a conversation about your idea
   - Visit [GitHub Discussions](https://github.com/beever-ai/beever-atlas/discussions)
   - Check if someone already suggested something similar
   - Provide use cases and examples

2. **Upvote Existing Requests** - Show what's important to you
   - Browse existing feature requests
   - Add reactions to show support
   - Comment with additional context

3. **Join Our Discord Community** - Chat with other users
   - Get real-time feedback from the community
   - Share your use cases and learn from others
   - Contribute ideas and help shape the roadmap

## How We Prioritize

Features are prioritized based on:

1. **Community Demand** - Number of upvotes and discussions
2. **Alignment with Mission** - Team-oriented automatic wiki generation
3. **Technical Feasibility** - Can we build it well?
4. **Resource Availability** - Do we have the capacity?
5. **Strategic Value** - Does it advance the v1.0 vision?
