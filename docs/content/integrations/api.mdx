# REST API

Beever Atlas provides a comprehensive REST API for building custom integrations, automations, and workflows.

<AutoTOC />

## Base URL

```
http://localhost:8000/api
```

For production, replace with your domain:
```
https://your-domain.com/api
```

## Authentication

### Development

No authentication required by default in development mode.

### Production

Configure reverse proxy authentication:

<Steps>
  <Step>
    **API Key Header**
    
    ```bash
    curl -H "Authorization: Bearer your-api-key" \\
      https://your-domain.com/api/channels
    ```
  </Step>
  <Step>
    **OAuth 2.0**
    
    Configure OAuth provider in reverse proxy (nginx, Traefik)
  </Step>
  <Step>
    **Mutual TLS**
    
    For enterprise deployments with mTLS
  </Step>
</Steps>

<Callout type="warning">
  Always implement authentication in production environments. Never expose the API publicly without authentication.
</Callout>

### Messages

List messages from a channel:

```bash
GET /channels/{channel_id}/messages?limit=100&before=1234567890.123456
```

**Parameters:**
- `limit` (number): Max messages (default: 100)
- `before` (string): Cursor for pagination
- `after` (string): Cursor for pagination
- `order` (string): `asc` or `desc` (default: `desc`)

**Response:**
```json
{
  "messages": [
    {
      "id": "1234567890.123456",
      "content": "What's the status of the API migration?",
      "author": "alice",
      "author_name": "Alice Johnson",
      "timestamp": "2024-01-15T10:30:00Z",
      "thread_id": null,
      "reactions": [{"emoji": "thumbsup", "count": 3}],
      "reply_count": 5
    }
  ],
  "next_cursor": "1234567889.123455"
}
```

## Knowledge Retrieval

### Ask

Ask a question with RAG:

```bash
POST /channels/{channel_id}/ask
```

**Request:**
```json
{
  "question": "What's our deployment process?",
  "conversation_id": "optional-conversation-id",
  "stream": true
}
```

**Response (streaming):**
```
data: {"type":"thinking","content":"Searching knowledge base..."}
data: {"type":"answer","content":"The deployment process involves..."}
data: {"type":"citation","type":"message","text":"...","permalink":"https://..."}
data: {"type":"done","confidence":0.89,"cost_usd":0.023}
```

**Response (non-streaming):**
```json
{
  "answer": "The deployment process involves...",
  "citations": [
    {
      "type": "message",
      "text": "Full message text...",
      "permalink": "https://slack.com/...",
      "timestamp": "2024-01-15T10:30:00Z"
    }
  ],
  "confidence": 0.89,
  "route": "qa",
  "cost_usd": 0.023
}
```

```bash
POST /search
```

**Request:**

```json
      "end": "2024-01-31T23:59:59Z"

  }

```

**Response:**
```json
{
  "results": [
    {
      "content": "We need to migrate the database...",
```

```python
import httpx
import json

# Ask a question with streaming
async def ask_question(channel_id: str, question: str):
    async with httpx.AsyncClient() as client:
        async with client.stream(
            "POST",
            f"http://localhost:8000/api/channels/{channel_id}/ask",
            json={"question": question, "stream": True}
        ) as response:
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    data = json.loads(line[6:])
                    if data["type"] == "answer":
                        print(data["content"], end="", flush=True)
                    elif data["type"] == "citation":
                        print(f"\n[Source: {data['permalink']}]")

# Usage
await ask_question("C0123456789", "What's our deployment process?")
```

```typescript
async function* askQuestion(
  channelId: string, 
  question: string
): AsyncGenerator<string> {
  const response = await fetch(
    `http://localhost:8000/api/channels/${channelId}/ask`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ question, stream: true })
    }
  );

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  if (!reader) return;

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const line = decoder.decode(value);
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      if (data.type === 'answer') {
        yield data.content;
      }
    }
  }
}

// Usage
for await (const chunk of askQuestion('C0123456789', 'Deployment process?')) {
  process.stdout.write(chunk);
}
```

```typescript
interface SearchResult {
  content: string;
  score: number;
  metadata: {
    channel_name: string;
    author: string;
    timestamp: string;
  };
}

interface SearchResponse {
  results: SearchResult[];
  total: number;
}

async function search(
  query: string, 
  channelId?: string
): Promise<SearchResponse> {
  const payload = { query };
  if (channelId) {
    (payload as any).channel_ids = [channelId];
  }

  const response = await fetch('http://localhost:8000/api/search', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload)
  });
  return response.json();
}

// Usage
const results = await search('database migration', 'C0123456789');
results.results.forEach(r => {
  console.log(`${r.score.toFixed(2)}: ${r.content.slice(0, 50)}...`);
});
```

## Rate Limiting

Default rate limits (configurable):

<Steps>
  <Step>
    **Per-IP Limits**
    
    - 100 requests per minute
    - Burst allowance: 20 requests
  </Step>
  <Step>
    **Per-Channel Limits**
    
    - 10 sync operations per hour
    - Prevents API abuse
  </Step>
  <Step>
  **Headers**
    
    ```http
    X-RateLimit-Limit: 100
    X-RateLimit-Remaining: 95
    X-RateLimit-Reset: 1642252800
    ```
  </Step>
</Steps>

## Error Handling

### Error Response Format

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Try again in 60 seconds.",
    "details": {
      "retry_after": 60
    }
  }
}
```

### Common Error Codes

| Code | HTTP | Description |
|------|------|-------------|
| `not_found` | 404 | Resource not found |
| `invalid_request` | 400 | Invalid request parameters |
| `unauthorized` | 401 | Missing or invalid authentication |
| `rate_limit_exceeded` | 429 | Rate limit exceeded |
| `internal_error` | 500 | Internal server error |

## Webhooks

Beever Atlas can send webhook notifications for events:

### Configure Webhook

```bash
POST /webhooks
```

**Request:**
```json
{
  "url": "https://your-domain.com/webhooks/beever",
  "events": ["sync.completed", "sync.failed"],
  "secret": "your-webhook-secret"
}
```

### Webhook Payload

```json
{
  "event": "sync.completed",
  "timestamp": "2024-01-15T12:00:00Z",
  "data": {
    "channel_id": "C0123456789",
    "messages_synced": 234,
    "duration_seconds": 12.3
  },
  "signature": "sha256=..."
}
```
