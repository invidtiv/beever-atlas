title: Telegram

Telegram ingestion has two live transports and one historical import path.

## Live Ingestion

### Polling

Use polling when Atlas is running locally, on a private network, or anywhere without a stable public HTTPS domain. Atlas calls Telegram Bot API `getUpdates` on a schedule and stores each new update before syncing it through the normal ingestion pipeline.

Polling uses the stored `offset` so processed updates are acknowledged only after they are durably saved. If Telegram reports that a webhook is already configured for the bot token, polling is blocked until the webhook is removed with `deleteWebhook`.

### Webhooks

Use webhooks when the bot bridge is deployed at a stable public HTTPS URL. Telegram sends updates to the bridge, and the bridge forwards those updates to the backend so webhook and polling updates share the same source-message store.

If you configure a Telegram `secret_token`, Telegram sends it in the `X-Telegram-Bot-Api-Secret-Token` header. Keep the token to letters, numbers, `_`, and `-`.

## Historical Messages

Telegram Bot API does not provide arbitrary historical chat backfill to bots. Bot ingestion starts from messages delivered after bot setup, and Telegram keeps pending bot updates only for a limited window.

Use Telegram Desktop JSON export for historical ingestion:

1. Export chat history from Telegram Desktop as JSON.
2. Upload `result.json` in Atlas file import.
3. Review the detected Telegram import and commit it.
4. Sync the imported Telegram channel.

For group chats, privacy/admin settings affect which messages the bot can see.
